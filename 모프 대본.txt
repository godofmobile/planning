1. 스케줄링(Scheduling)

정의 : 컴퓨터 시스템을 구성하고 있는 주기억 장치, 입출력 장치, 처리시간 등의 시스템 자원을 언제 배분할 것인가를 결정하는 프로그램 기능

어플적용 : 예를 들면 버튼 안에 이벤트를 넣었다고 가정할 때, 그 이벤트는 버튼이 눌릴때만 실행이 되는데, 스케줄링 기법을 사용하면 프로세스가 계속 돌면서 체크를 한다. 그 때 이벤트 실행이 필요하다면 버튼이 계속 눌려진다. 이러한 방식을 우리 팀에서는 훈남캐릭터의 계속되는 움직임, 충돌, 게임 시작 후 시간 감소 에서 구현했다.

2. 랜덤포지션(Random Position)

어플적용 : 훈남캐릭터들이 랜덤하게 나오게 하기 위해 만들어준 함수이다.
디스플레이를 x,y 좌표로 두고 훈남 캐릭터들 좌표를 스케줄링 기법을 사용하여 랜덤하게 나오게 된다.

3. 데이터베이스(Database)

어플적용 : db file 생성 -> db table 생성 -> db data insert -> db data load -> db close

sqlite3을 사용하여 게임정보를 저장했다가 불러온다. 우리 어플에서는 stage와 coin이 적용된다.

4. 게임 이미지 변환(Animation)

어플적용 : 훈남캐릭터를 클릭하면 오징어캐릭터로 바뀌는데, 이것은
새로운 이미지를 원래 있던 이미지 위치에 두고 원래 이미지는 좌표를 찍어 멀리 날리는 식으로 개발했다.

5. 음향(BGM)

어플적용 : 게임을 시작하면 왕벌의 비행이 BGM으로 깔린다.
훈남캐릭터를 클릭해서 오징어캐릭터로 바뀔 때 소리가 나온다.
미션 성공이나 실패시 소리가 나온다.


6. 게임 난이도

세단계로 나뉘어져있다.

슬라이드를 통해서 스토리보드가 바뀔 것이라는 것을 보여준다.

8. Coupon data

단국대학교 근처 헤어숍, 옷가게를 찾아가서 사장님과 얘기를 해본 뒤 입권되는 상권이 있다면 홍보를 해주고 쿠폰을 받아올 것이다.
